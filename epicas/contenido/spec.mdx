# Spec: Contenido (AI Content Generator)

## Fuentes

- [Especificaciones de Actividades (lineamientos de calidad)](/epicas/contenido/specs-actividades)
- Bibliografía pedagógica: Anderson & Krathwohl (2001), Haladyna, Downing & Rodriguez (2002), Butler (2018).

## Flujos principales

### Flujo 1: Indexación de fuentes

<Steps>
  <Step title="Carga de fuentes bibliográficas">
    Se cargan fuentes bibliográficas (PDF/texto) para una materia.
  </Step>
  <Step title="Indexación mediante LLM">
    Se ejecuta el proceso de indexación: el sistema recorre cada párrafo del documento.
  </Step>
  <Step title="Asignación a concepto">
    Cada párrafo es asignado a un **concepto** dentro de la estructura curricular de la materia (módulo > unidad > concepto).
  </Step>
  <Step title="Contenido disponible">
    El contenido indexado queda disponible para generación de actividades.
  </Step>
</Steps>

### Flujo 2: Generación de actividades

<Steps>
  <Step title="Solicitud a demanda">
    Se solicita la generación de actividades a demanda (por tipo, concepto, nivel cognitivo).
  </Step>
  <Step title="Generación siguiendo lineamientos">
    El generador produce la actividad siguiendo los lineamientos de calidad y el contenido indexado.
  </Step>
  <Step title="Control de calidad pre-docente">
    Se ejecuta el checklist automatizado.
  </Step>
  <Step title="Incorporación a la base de datos">
    Si la actividad pasa el control, se incorpora a la base de datos de TUNI. Opcionalmente, se generan imágenes complementarias.
  </Step>
</Steps>

### Flujo 3: Control de calidad

<Steps>
  <Step title="Checklist automatizado">
    Cada actividad generada pasa por un checklist automatizado antes de ser aceptada.
  </Step>
  <Step title="Verificación de criterios">
    Se verifican: alineación con objetivo de aprendizaje, nivel cognitivo, claridad, completitud de campos, ausencia de frameworks pedagógicos expuestos, localización lingüística, fidelidad a la fuente.
  </Step>
  <Step title="Resultado">
    Actividades que no pasan el control son marcadas para revisión o regeneración.
  </Step>
</Steps>

```
Carga de fuentes (PDF/texto)
  --> Indexación (LLM: párrafo --> concepto)
  --> Contenido indexado (por módulo/unidad/concepto)
  --> Generación de actividades (a demanda, por tipo)
  --> Control de calidad pre-docente
  --> Base de datos TUNI
```

## Componentes

### Indexador

- Recibe PDF/texto como input.
- Utiliza LLM para analizar párrafo por párrafo.
- Asigna cada párrafo a un concepto de la estructura curricular (módulo > unidad > concepto).
- Output: contenido indexado disponible para generación.

### Generador de actividades

- Recibe: concepto, tipo de actividad solicitado, contenido indexado relevante.
- Produce: actividad completa con todos los campos requeridos según el tipo (enunciado, opciones, respuesta correcta, feedback/rúbrica).
- Respeta los lineamientos de calidad definidos en la [Especificación de Actividades](/epicas/contenido/specs-actividades).

### Generador de imágenes

- Genera imágenes complementarias para actividades que lo requieran.
- Pendiente de especificación detallada.

### Validador (control de calidad pre-docente)

- Ejecuta checklist automatizado sobre cada actividad generada.
- Verifica: completitud de campos, ausencia de frameworks pedagógicos, localización lingüística, fidelidad a la fuente, nivel cognitivo adecuado.
- Marca actividades que no pasan para revisión o regeneración.

## Reglas de contenido obligatorias

<Warning>
  Toda actividad generada debe cumplir las siguientes reglas (detalle completo en la [Especificación de Actividades](/epicas/contenido/specs-actividades)):
</Warning>

1. **Prohibido mencionar frameworks pedagógicos:** no incluir referencias a taxonomía de Bloom, niveles cognitivos, diseño instruccional ni conceptos de pedagogía que sean parte del proceso interno de generación.
2. **Prohibido usar terminología extranjerizada:** no usar anglicismos crudos ni terminología de otros sistemas. Adaptar siempre al español argentino y al sistema institucional local.
3. **Prohibido generar contenido fuera de la disciplina:** los ejemplos, escenarios y preguntas deben ser estrictamente de la materia correspondiente.
4. **Prohibido incluir la respuesta en el enunciado:** el enunciado debe estar limpiamente separado de las opciones y la respuesta correcta.
5. **Prohibido referenciar material sin contexto:** si una actividad menciona un caso, fallo o documento específico, debe incluir el extracto o descripción suficiente para que el alumno pueda responder.

## Consideraciones técnicas

- **Fidelidad a la fuente:** toda actividad se deriva estrictamente del material bibliográfico cargado. No se inventan escenarios, datos ni ejemplos sin respaldo.
- **Simplicidad y claridad:** enunciados concisos, sin ambigüedad, longitud acotada.
- **Localización lingüística:** español rioplatense/argentino, terminología del sistema argentino.
- **Autosuficiencia:** toda actividad es autocontenida, respondible con la información proporcionada.
- **Invisibilidad del framework pedagógico:** nunca exponer al alumno conceptos como "Bloom", "carga cognitiva" o "diseño instruccional".
- **Nivel adecuado al destinatario:** calibrar complejidad al año de cursada y perfil del alumno.
