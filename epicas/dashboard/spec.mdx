## Objetivo

El **Dashboard** es el panel principal del estudiante para:

- Acceder rápidamente a las funcionalidades clave: **Clase**, **Consultas** y **Evaluaciones**.
- **Visualizar progreso** y reforzar motivación mediante métricas simples.

## Flujos principales

### 1) Acceso → Materias

**Pantalla: Login**

- Campos: correo + contraseña.
- CTA: "Iniciar sesión".

**Pantalla: Materias**

- Saludo ("¡Hola, [nombre]!") + pregunta ("¿Con qué materia comenzamos?").
- Cards por materia con:
  - Nombre de materia.
  - Próximo parcial (ej. "en 10 días / 20 días / 28 días") si existe el dato.
  - Etiqueta "Recomendada" si corresponde.
  - Unidad actual (ej. "Principios del Derecho", "Teoría del Delito", etc.).
  - Indicador de progreso/dominio (porcentaje o visual).

**Resultado del flujo**

- Seleccionar una materia navega al **Dashboard de materia**.

### 2) Dashboard de materia → Empezar a estudiar

- El dashboard de la materia se accede desde la **sidebar** al seleccionar una materia específica y muestra únicamente información asociada a ella.
- También puede accederse desde el **CTA de cierre de una clase** ("Ir a mi materia").

**Entry point principal**

- Banner superior "**Clase recomendada**" (badge "Aprendizaje adaptativo") con CTA "**Empezar a estudiar**".

**Comportamiento**

- Al click, el usuario es redirigido a una **clase con el tutor** correspondiente a la **unidad priorizada por el sistema**, determinada según su **nivel de dominio**.

### 3) Dashboard de materia → "Tengo una consulta"

**Entry point**

- Botón secundario en la home, junto a "Empezar a estudiar".

**Comportamiento**

- Abre un **side panel** (no ventana nueva).
- No muestra historial de consultas anteriores.
- Al abrir: se muestra el **último chat** + botón "Nueva consulta".

**Derivación a clase**

- La consulta puede transformarse en clase:
  - El sistema pregunta: "¿Querés hacer una clase sobre esto?"
  - Si el usuario acepta, deriva al flujo de clase con ese tema.
- Si la materia no es clara:
  - El sistema puede preguntar la materia (pendiente exactitud: preguntar vs deducir).

### 4) Dashboard de materia → Unidad (detalle / side sheet)

**Entry points**

- En cada unidad (ítem dentro de "Contenidos"), en estado hover aparecen botones: **"Estudiar"** y **"Ver detalle"**.

**Reglas de UI (según diseño)**

- El título de la unidad admite un **máximo de una línea**.
- Si excede el límite, se **trunca con puntos suspensivos** (`…`).
- Los botones **"Estudiar"** y **"Ver detalle"**:
  - Se muestran al hacer **hover** sobre la unidad.
  - Permanecen **alineados a la derecha** y **centrados verticalmente** dentro del contenedor.

**Acción: Estudiar**

- Redirige a una **clase con el tutor** sobre dicha unidad.

**Acción: Ver detalle**

- Abre un **side sheet** (overlay) que se despliega **de derecha a izquierda** ocupando el espacio de los módulos/métricas.
- Debe aplicar un **background** (overlay oscuro) y permitir cierre:
  - Click por fuera del side sheet.
  - Click en el ícono **"X"**.

**Contenido del side sheet (Detalle unidad)**

- Contexto de módulo (ej. "1. Fundamentos y metodología…").
- Título y breve descripción de la unidad.
- Indicador de progreso: donut "**Dominio de la unidad**".
- Sección "**Dominio de temas**" (lista de temas con barra + porcentaje).
- CTA primario: "**Estudiar esta unidad**".

**Decisiones de UX**

- Se elimina el botón "Repasar unidad".
- "Tus chats" pasa a llamarse "Clases de la unidad".

### 5) Cierre de clase → "Revisar desafío"

- En cierre de clase aparece CTA "Revisar desafío".
- Navega a revisión pregunta por pregunta con feedback.
- Controles: "Siguiente" / "Atrás".

## Evaluaciones (MVP redefinido)

### Principios

- **No bloquean el avance** → son **optativas**.
- Se mantienen **solo a nivel módulo** (no por unidad) para MVP.
- Se pueden realizar múltiples evaluaciones del mismo módulo.
- Durante evaluación: **sin feedback en tiempo real** (simula examen real).

### Cierre de evaluación (pantalla)

- Nota obtenida.
- Gráfico por unidad (puntaje por unidad del módulo).
- "Áreas fuertes" / "Áreas de mejora".
- Comparación vs evaluación anterior (si existe).
- CTAs:
  - "Ver detalle" → navegar pregunta por pregunta con feedback.
  - "Aprender de mis errores" → derivar a clase de refuerzo específica.
  - "Ir a mi materia" → cerrar.

### Historial

- Sección "Ver evaluaciones" con historial (side panel), agrupado por módulo.

## Pantallas / Módulos

- **Login** (si aplica al entorno; puede existir SSO/Canvas en integraciones).
- **Materias** (home con cards por materia).
- **Dashboard de materia** (home dentro de una materia: banner, progreso, métricas, contenidos).
- **Side panel de unidad** (detalle de módulo/unidad + lista de clases).
- **Side panel de consultas** ("Tengo una consulta").
- **Cierre de clase** (métricas + revisión de desafío).
- **Evaluación** (flujo de evaluación + cierre + historial).

## Componentes

### Global / Layout

- Header con:
  - Racha diaria (persistente, estilo Duolingo).
  - Indicadores de moneda/XP (según diseño).
  - Avatar/menú de usuario.
- Sidebar (iconos) para navegación global.

### Materias

- `MateriaCard`
  - `RecommendedBadge`
  - `NextExamPill` (si existe dato)
  - `CurrentUnitRow`
  - `ProgressIndicator` (dominio)

### Dashboard de materia

- `SubjectHeader` (título materia)
- `RecommendedClassBanner`
  - Badge "Aprendizaje adaptativo"
  - Título: "Clase recomendada"
  - Descripción: "Diseñada para ayudarte a dar el próximo paso."
  - CTA: "Empezar a estudiar"

- `SubjectProgressCard`
  - **Con data**: donut "Dominio" + lista "Unidades con mayor progreso"
  - **Empty state**: ver sección "Estados" (cta "Empezar a estudiar")

- `TopUnitsProgressList` (unidades con mayor progreso)
- `WeeklyMetricsCard` ("Esta semana")
  - `ClassesCompletedMetric` (ej. 5)
  - `HoursStudiedMetric` (ej. 3)
  - `WeeklyProgressMetric` (ej. 10%)

- `ContentsModuleList`
  - `ModuleSection` (por módulo/materia)
  - `UnitRow` (default/hover)
  - `EvaluationModuleRow` (ej. "Evaluación Módulo 1")

- (Legacy) `SubjectBanner` (compacto, sin imagen por ahora)
- (Legacy) `SubjectProgressChart` (se mueve arriba por visibilidad)
- `MasteryDonutSubject` vs `MasteryDonutUnit` (diferenciación visual)
- `MetricsRow`
  - `ClassesCompletedMetric`
  - `HoursStudiedMetric`
  - `WeeklyProgressMetric`
- `PrimaryCTAStartStudying`
- `SecondaryCTAConsult`

### Side panels

- `UnitDetailSideSheet` (overlay "Detalle unidad")
- `ConsultSidePanel`
- `EvaluationHistorySidePanel`

### Cierres

- `ClassClosingScreen` + `StreakWidget` + `ReviewChallengeCTA`
- `EvaluationClosingScreen` + `QuestionReviewNavigator`

## Consideraciones técnicas

- Las reglas de nomenclatura UX definen: "Concepto" se muestra como "Tema", "Diagnóstico" como "Evaluación".
- La gamificación mínima (racha diaria) es persistente en header y en cierre de clase.
- Las evaluaciones no bloquean avance (optativas), solo a nivel módulo para MVP, sin feedback en tiempo real (simula examen).
- Se contemplan estados: empty (sin progreso), loading/skeleton, error (fallo de red / sin permisos).
