## Problema

TUNI fue construido desktop-first. El público objetivo (estudiantes universitarios, 18–25 años) es mayoritariamente mobile-first. Si la experiencia no funciona bien en el celular, la barrera de entrada para el uso diario es demasiado alta. Mobile es el prerequisito para impactar DAU.

## Objetivos

- **Adaptar la experiencia de clase** para que funcione nativamente en mobile.
- **Eliminar la fricción** de tener que usar desktop para estudiar.
- **Habilitar uso diario** desde el dispositivo principal del estudiante.
- **Explorar canales alternativos** de interacción (WhatsApp, PWA).

## Enfoque técnico

### PWA como punto de partida

Ya existe una base técnica (ticket DASH-011) para Progressive Web App. El enfoque es extender esta base para lograr una experiencia mobile completa:

- Responsive real (no solo "se ve" en mobile, sino que funciona bien)
- Instalable como app desde el navegador
- Push notifications (necesarias para [Comunidad](/epicas/comunidad/overview))

### Canal WhatsApp (exploración)

Idea en exploración: que Ada esté disponible vía WhatsApp para interacciones rápidas. No reemplaza la experiencia completa de clase, pero puede ser un canal complementario para:
- Consultas rápidas
- Recordatorios de estudio
- Notificaciones de desafíos

<Note>
  El canal WhatsApp es exploratorio y no está confirmado como parte del alcance. Requiere evaluación técnica y de producto.
</Note>
