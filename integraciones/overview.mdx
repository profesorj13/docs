# Integraciones

> Conexiones con sistemas externos (LMS, APIs) para que TUNI funcione dentro del ecosistema tecnológico de las instituciones educativas.

## Problema

Los estudiantes ya tienen un entorno de estudio principal (Canvas LMS). Si TUNI vive únicamente como plataforma separada, la fricción de adopción es alta y se pierde el contexto de lo que el alumno está estudiando en cada momento. Se necesita integrar TUNI en los sistemas que los alumnos ya usan.

## Alcance MVP

### Incluye

- Integración con **Canvas LMS** (widget flotante, autenticación LTI/OAuth, sincronización de cursos).
- Piloto con 3 materias de Siglo 21.

### No incluye

<Warning>
  Los siguientes elementos están fuera del alcance MVP:
  - Otros LMS (Moodle, Blackboard, etc.).
  - Integración bidireccional de calificaciones.
  - Sincronización con Calendar o Assignments API de Canvas.
</Warning>

## Principios de diseño

- **Integración no invasiva:** TUNI se adapta al entorno existente del alumno, sin alterar la experiencia del LMS.
- **Registro diferido:** permitir interacción inicial sin autenticación para maximizar engagement.
- **Contexto automático:** detectar materia, módulo y contenido actual sin intervención del usuario.
- **Arquitectura extensible:** diseñar para Canvas hoy, pero con capacidad de agregar otros LMS mañana.
